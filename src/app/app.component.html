Old vs new svg binding syntax demo

This is the old way. It does not work as of Angular 9

Approach taken from this (old) StackOverflow example

  • The svg element receives default sizing instead of the supplied pixel values when using the [attr.width.px]="var" syntax.
  • The circle element does not get rendered due to supplying the radius using the [attr.r.px]="r" syntax.

This is the new way.

  • The svg element receives the supplied pixel values when using the [attr.width]="var" syntax.
  • The circle element gets rendered when supplying the radius using the [attr.r]="r" syntax.
